# 步骤6：测试框架

## 6.1 知识点概述

测试是保证代码质量的重要手段，本节将学习如何使用 pytest 进行单元测试和集成测试。

### 核心概念

- **单元测试**：测试单个函数或类
- **集成测试**：测试多个组件的交互
- **端到端测试**：测试完整功能流程
- **测试覆盖**：代码被测试的程度
- **测试夹具**：测试前的准备工作

### 技术栈

- **pytest**：Python 测试框架
- **pytest-asyncio**：异步测试支持
- **httpx**：异步 HTTP 客户端
- **pytest-cov**：测试覆盖率

### Q1: 测试数据库连接失败

**原因**：
- 测试数据库不存在
- 数据库连接参数错误

**解决方案**：
```python
# 确保测试数据库存在（MySQL）
import asyncio
from sqlalchemy import text

async def create_test_database():
    """创建测试数据库"""
    # 连接到默认数据库
    engine = create_async_engine(
        f"mysql+aiomysql://{user}:{password}@{host}:{port}/mysql"
    )
    
    async with engine.connect() as conn:
        # 创建测试数据库
        await conn.execute(text(f"CREATE DATABASE IF NOT EXISTS test_{database}"))
    
    await engine.dispose()
```

### Q3: 测试数据污染

**原因**：
- 测试之间共享数据
- 测试后未清理数据

**解决方案**：
```python
@pytest_asyncio.fixture(scope="function")
async def db_session():
    """每个测试使用独立的数据库会话"""
    async with test_engine.begin() as conn:
        await conn.run_sync(Base.metadata.create_all)
    
    async with test_async_session_factory() as session:
        yield session
    
    async with test_engine.begin() as conn:
        await conn.run_sync(Base.metadata.drop_all)
```
