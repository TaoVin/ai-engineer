## 8.2 ASGI 协议

### 什么是 ASGI

ASGI（Asynchronous Server Gateway Interface）是 Python 异步 Web 服务器和应用之间的标准通信协议。它由 Django 团队提出，是 WSGI 的异步扩展版本。

```
┌─────────────────────────────────────────────────────────┐
│                      ASGI 服务器                         │
│                   （Uvicorn / Hypercorn）                │
│                                                         │
│  职责：                                                   │
│  1. 监听 TCP 端口，接受客户端连接                          │
│  2. 解析 HTTP 请求（或 WebSocket 帧）                     │
│  3. 将请求封装为 ASGI scope / receive / send              │
│  4. 调用应用的 __call__ 方法                              │
│  5. 将应用的响应发送回客户端                               │
└────────────────────────┬────────────────────────────────┘
                         │
                         │  ASGI 协议（scope, receive, send）
                         ▼
┌─────────────────────────────────────────────────────────┐
│                      ASGI 应用                           │
│              （FastAPI / Starlette / Django）             │
│                                                         │
│  职责：                                                   │
│  1. 接收 scope（请求元数据）                               │
│  2. 从 receive 读取请求体                                 │
│  3. 处理业务逻辑                                          │
│  4. 通过 send 发送响应                                    │
└─────────────────────────────────────────────────────────┘
```

### ASGI 应用的本质

一个 ASGI 应用就是一个遵循特定协议的 Python 对象。最简单的 ASGI 应用：

```python
async def app(scope, receive, send):
    """最简单的 ASGI 应用"""
    assert scope['type'] == 'http'

    # 发送 HTTP 响应头
    await send({
        'type': 'http.response.start',
        'status': 200,
        'headers': [[b'content-type', b'text/plain']],
    })
    # 发送 HTTP 响应体
    await send({
        'type': 'http.response.body',
        b'Hello, World!',
    })
```

FastAPI 的 `FastAPI()` 类本质上就是一个 ASGI 应用，它实现了 `__call__(self, scope, receive, send)` 方法。

### WSGI vs ASGI

| 特性 | WSGI | ASGI |
|------|------|------|
| 协议类型 | 同步 | 异步 |
| 并发模型 | 一个请求占用一个线程 | 单线程事件循环，I/O 时切换 |
| WebSocket | 不支持 | 原生支持 |
| HTTP/2 | 不直接支持 | 支持 |
| 适用框架 | Flask、Django（传统） | FastAPI、Starlette、Django 3.0+ |
| 服务器 | Gunicorn、uWSGI | Uvicorn、Hypercorn、Daphne |
| 性能（I/O 密集） | 一般 | 高 |

### 请求在 ASGI 中的流转

```
1. 客户端发起 HTTP 请求
        │
2. Uvicorn 接收 TCP 连接，解析 HTTP 协议
        │
3. Uvicorn 构造 ASGI scope（包含 method、path、headers 等）
        │
4. Uvicorn 调用 app(scope, receive, send)
        │
5. FastAPI 根据 path 匹配路由
        │
6. 执行依赖注入、参数校验
        │
7. 调用端点函数（如 def get_user():）
        │
8. 端点函数返回数据
        │
9. FastAPI 序列化响应，调用 send() 发回 Uvicorn
        │
10. Uvicorn 将响应写入 TCP 连接，返回给客户端
```

## 8.3 Uvicorn 核心概念

### 什么是 Uvicorn

Uvicorn 是一个轻量级、高性能的 ASGI 服务器，是 FastAPI 官方推荐的运行服务器。

### 核心依赖

| 组件 | 作用 | 说明 |
|------|------|------|
| `uvloop` | 事件循环 | 基于 libuv（Node.js 底层），比 asyncio 默认循环快 2-4 倍 |
| `httptools` | HTTP 解析 | 基于 http-parser（Node.js 底层），C 实现，性能极高 |
| `h11` | HTTP/1.1 协议 | 纯 Python 实现，httptools 的备选方案 |
| `websockets` | WebSocket | WebSocket 协议支持 |

安装时的区别：

```bash
# 仅安装 uvicorn（使用 h11 纯 Python HTTP 解析）
pip install uvicorn

# 安装 uvicorn[standard]（使用 httptools C 实现 + uvloop，性能更好）
pip install uvicorn[standard]
```

**推荐使用 `uvicorn[standard]`**，当前项目的 `pyproject.toml` 已配置此依赖。

## 8.4 `app.main:app` 路径解析

### 格式说明

```
uvicorn app.main:app
         ────┬─── ─┬─
             │     │
             │     └── 变量名：模块内的 Python 变量名
             └── 模块路径：Python 导入路径（用 . 分隔，对应文件系统 /）
```

### 转换规则

```
app.main:app
    │
    ├── 模块路径 app.main
    │       │
    │       ├── app/          → 项目中的 app 目录
    │       ├── /             → 对应 Python 的 .
    │       └── main          → 对应 main.py 文件
    │
    └── 最终路径 → app/main.py 文件中的 app 变量
```

### 实际例子

| 启动参数 | 对应文件 | 对应变量 |
|---------|---------|---------|
| `app.main:app` | `app/main.py` | `app` 变量 |
| `app.main:create_app` | `app/main.py` | `create_app` 函数 |
| `main:app` | `main.py`（根目录） | `app` 变量 |
| `src.app.main:application` | `src/app/main.py` | `application` 变量 |

### 为什么需要在 main.py 中暴露变量

Uvicorn 启动时会执行类似以下操作：

```python
# Uvicorn 内部逻辑（简化）
import importlib

module_path, app_name = "app.main", "app"
module = importlib.import_module(module_path)  # 导入 app/main.py
app = getattr(module, app_name)                # 获取 app 变量
# 然后以 app 为 ASGI 应用启动服务器
```

如果 `main.py` 中没有 `app` 变量，`getattr` 会抛出 `AttributeError`。

## 8.5 Uvicorn 命令行配置

### 基本启动命令

```bash
# 最简启动（默认 127.0.0.1:8000）
uvicorn app.main:app

# 指定主机和端口
uvicorn app.main:app --host 0.0.0.0 --port 8001

# 开启热重载（开发模式，文件变更自动重启）
uvicorn app.main:app --reload

# 指定工作进程数（生产模式，不能与 --reload 同时使用）
uvicorn app.main:app --workers 4
```

### 完整参数列表

#### 网络参数

```bash
--host 0.0.0.0          # 监听地址（默认 127.0.0.1）
                        # 127.0.0.1 = 仅本机可访问
                        # 0.0.0.0  = 允许外部访问（生产环境常用）

--port 8001             # 监听端口（默认 8000）

--uds /tmp/uvicorn.sock # Unix Domain Socket（替代 host:port，Nginx 反代常用）
```

**`--host` 的区别**：

```
--host 127.0.0.1（默认）               --host 0.0.0.0
┌─────────────┐                       ┌─────────────┐
│   服务器     │                       │   服务器     │
│             │                       │             │
│  ┌───────┐  │                       │  ┌───────┐  │
│  │Uvicorn│  │ ← 仅本机可访问        │  │Uvicorn│  │ ← 本机 + 外部均可访问
│  └───┬───┘  │                       │  └───┬───┘  │
│      │      │                       │      │      │
│  127.0.0.1  │                       │  0.0.0.0   │
└─────────────┘                       └─────────────┘
  本机浏览器 → ✓ 可访问                  本机浏览器 → ✓ 可访问
  其他机器   → ✗ 不可访问                其他机器   → ✓ 可访问
```

#### 开发模式参数

```bash
--reload                # 开启热重载（文件变更自动重启）
--reload-dir app        # 监视的目录（默认当前目录）
--reload-dir config     # 可多次指定，监视多个目录
--reload-delay 0.25     # 重载延迟（秒），避免频繁重启
```

#### 生产模式参数

```bash
--workers 4             # Worker 进程数（推荐 CPU 核心数 × 2 + 1）
                        # 注意：不能与 --reload 同时使用
```

#### 日志参数

```bash
--log-level info        # 日志级别：debug / info / warning / error / critical
--access-log            # 启用访问日志（记录每个请求）
--access-log-format %%(asctime)s %%(client_addr)s %%(request_line)s %%(status_code)s
                        # 访问日志格式
```

#### SSL 参数（HTTPS）

```bash
--ssl-keyfile /path/to/key.pem      # SSL 私钥文件
--ssl-certfile /path/to/cert.pem    # SSL 证书文件
```

#### 工厂函数参数

```bash
--factory               # 指定的入口是工厂函数（会自动调用获取实例）
```

### 参数组合示例

```bash
# 开发环境：热重载 + 详细日志
uvicorn app.main:app \
    --host 0.0.0.0 \
    --port 8001 \
    --reload \
    --reload-dir app \
    --reload-delay 0.5 \
    --log-level debug \
    --access-log

# 生产环境：多 Worker + 访问日志
uvicorn app.main:app \
    --host 0.0.0.0 \
    --port 8001 \
    --workers 4 \
    --log-level warning \
    --access-log \
    --access-log-format '%(asctime)s %(client_addr)s "%(request_line)s" %(status_code)s'
```

## 8.6 Uvicorn Python 代码配置

### 为什么需要代码配置

某些场景下，命令行参数不够灵活：

- 需要根据环境变量动态计算参数
- 需要在启动前执行初始化逻辑
- 需要集成到其他 Python 脚本中

### 方式一：在 main.py 中直接启动

在 `app/main.py` 底部添加：

```python
import uvicorn

def create_app() -> FastAPI:
    app = FastAPI(...)
    return app

app = create_app()

# 直接运行 python app/main.py 启动服务
if __name__ == "__main__":
    uvicorn.run(
        "app.main:app",       # 字符串形式，支持热重载
        host="0.0.0.0",
        port=8001,
        reload=True,
        log_level="info",
    )
```

启动方式：

```bash
python app/main.py
```

### `uvicorn.run()` 参数详解

```python
uvicorn.run(
    # ===== 应用入口 =====
    "app.main:app",          # 字符串形式（推荐，支持 reload）
    # 或
    app,                      # 直接传递 app 对象（不支持 reload）

    # ===== 网络 =====
    host="0.0.0.0",          # 监听地址
    port=8001,               # 监听端口
    uds=None,                # Unix Socket 路径
    fd=None,                 # 已打开的文件描述符

    # ===== Worker =====
    workers=None,            # Worker 进程数（None=单进程）
    loop="auto",             # 事件循环：auto/asyncio/uvloop
    http="auto",             # HTTP 协议实现：auto/h11/httptools
    ws="auto",               # WebSocket 实现：auto/websockets/none

    # ===== 开发 =====
    reload=True,             # 热重载
    reload_dirs=["app"],     # 监视目录列表
    reload_delay=0.25,       # 重载延迟

    # ===== 日志 =====
    log_level="info",        # 日志级别
    access_log=True,         # 启用访问日志

    # ===== SSL =====
    ssl_keyfile=None,        # SSL 私钥
    ssl_certfile=None,       # SSL 证书

    # ===== 生命周期 =====
    lifespan="auto",         # 生命周期处理器：auto/on/off
)
```

### 方式二：使用 `if __name__ == "__main__"` 模式

这是最推荐的代码配置方式，因为它同时支持两种启动方式：

```python
# app/main.py

import sys
from pathlib import Path

ROOT_DIR = Path(__file__).resolve().parent.parent
if str(ROOT_DIR) not in sys.path:
    sys.path.insert(0, str(ROOT_DIR))

import uvicorn
from fastapi import FastAPI
from app.config.settings import setting


def create_app() -> FastAPI:
    """应用工厂函数"""
    app = FastAPI(
        title=setting.TITLE,
        version=setting.VERSION,
        summary=setting.SUMMARY,
        description=setting.DESCRIPTION,
        docs_url=setting.DOCS_URL,
        redoc_url=setting.REDOC_URL,
        root_path=setting.ROOT_PATH,
        debug=setting.DEBUG,
    )
    return app


app = create_app()


if __name__ == "__main__":
    # 直接运行：python app/main.py
    uvicorn.run(
        "app.main:app",
        host=setting.SERVER_HOST,
        port=setting.SERVER_PORT,
        reload=setting.DEBUG,
        log_level="debug" if setting.DEBUG else "info",
    )
```

这样既可以通过 `python app/main.py` 直接启动，也可以通过 `uvicorn app.main:app` 命令启动。

### 方式三：独立启动脚本

创建 `run.py`（项目根目录）：

```python
#!/usr/bin/env python3
"""
应用启动脚本
用法：python run.py
"""

import uvicorn
from app.config.settings import setting


def main():
    uvicorn.run(
        "app.main:app",
        host=setting.SERVER_HOST,
        port=setting.SERVER_PORT,
        reload=setting.DEBUG,
        log_level="debug" if setting.DEBUG else "info",
        access_log=True,
    )


if __name__ == "__main__":
    main()
```

启动：

```bash
python run.py
```

## 8.7 应用工厂模式详解

### 为什么用工厂函数

```python
# 不推荐：直接创建
app = FastAPI()     # 模块导入时立即执行，难以测试

# 推荐：工厂函数
def create_app():
    app = FastAPI()  # 调用时才创建，可控、可测试
    return app
app = create_app()   # 模块级别暴露，供 uvicorn 使用
```

工厂函数的优势：

1. **便于测试**：测试时创建独立实例，不污染全局状态
2. **延迟初始化**：app 在函数调用时才创建
3. **统一管理**：路由、中间件、事件等注册集中在一处
4. **多实例**：可以创建不同配置的 app（如测试配置、生产配置）

## 8.8 启动脚本编写

### 设计思路

项目有 `.env.dev` 和 `.env.prod` 两个配置文件，包含 `SERVER_HOST`、`SERVER_PORT`、`DEBUG` 等配置。启动脚本需要：

1. 接收环境参数（dev / prod）
2. 自动读取对应的 `.env.{env}` 文件
3. 将读取到的配置传递给 uvicorn / gunicorn

```
.env.dev                        scripts/start.sh                 uvicorn
┌──────────────┐               ┌──────────────────┐           ┌──────────────┐
│SERVER_HOST=  │   read_env()  │  读取配置         │  --host   │              │
│  localhost   │──────────────→│  SERVER_HOST     │──────────→│   Uvicorn    │
│SERVER_PORT=  │               │  SERVER_PORT     │  --port   │   Server     │
│  8001        │               │  DEBUG           │  --reload │              │
│DEBUG=True    │               │                  │           │              │
└──────────────┘               └──────────────────┘           └──────────────┘
```

### 统一启动脚本

创建 `scripts/start.sh`，一个脚本同时支持开发和生产环境：

```bash
#!/bin/bash
# ============================================
# 项目启动脚本
# 用法：
#   ./scripts/start.sh              # 默认 dev 环境
#   ./scripts/start.sh dev          # 开发环境
#   ./scripts/start.sh prod         # 生产环境
# ============================================

set -euo pipefail

# ---------- 颜色定义 ----------
RED='\033[0;31m'
GREEN='\033[0;32m'
YELLOW='\033[0;33m'
CYAN='\033[0;36m'
NC='\033[0m'

info()  { echo -e "${GREEN}[INFO]${NC}  $*"; }
warn()  { echo -e "${YELLOW}[WARN]${NC}  $*"; }
error() { echo -e "${RED}[ERROR]${NC} $*"; exit 1; }

# ---------- 路径设置 ----------
SCRIPT_DIR="$(cd "$(dirname "$0")" && pwd)"
PROJECT_DIR="$(cd "$SCRIPT_DIR/.." && pwd)"
cd "$PROJECT_DIR"

# ---------- 解析环境参数 ----------
ENV="${1:-dev}"

if [[ "$ENV" != "dev" && "$ENV" != "prod" ]]; then
    error "不支持的环境: $ENV（可选: dev / prod）"
fi

ENV_FILE=".env.${ENV}"

# ---------- 检查 .env 文件 ----------
if [ ! -f "$ENV_FILE" ]; then
    error "未找到配置文件: $ENV_FILE"
fi

# ---------- 从 .env 文件读取配置 ----------
# 用法: read_env KEY 默认值
# 从 .env 文件中提取指定 key 的值，处理空格、引号、注释
read_env() {
    local key="$1"
    local default="${2:-}"
    local value

    # 提取 key = value，去除行尾注释，去除首尾空格和引号
    value=$(grep -E "^${key}\s*=" "$ENV_FILE" \
        | head -1 \
        | sed "s/^${key}\s*=\s*//" \
        | sed 's/\s*#.*$//' \
        | sed 's/^["'\'']//' \
        | sed 's/["'\'']$//' \
        | xargs)

    # 如果为空则使用默认值
    if [ -z "$value" ]; then
        echo "$default"
    else
        echo "$value"
    fi
}

# 读取关键配置
SERVER_HOST=$(read_env SERVER_HOST "0.0.0.0")
SERVER_PORT=$(read_env SERVER_PORT "8001")
DEBUG=$(read_env DEBUG "True")
LOGGER_LEVEL=$(read_env LOGGER_LEVEL "INFO")

# 导出环境变量，供 Python settings.py 读取
export ENVIRONMENT="$ENV"

# ---------- 激活虚拟环境 ----------
if [ -f ".venv/bin/activate" ]; then
    source .venv/bin/activate
    info "已激活虚拟环境"
else
    warn "未找到虚拟环境，使用系统 Python"
fi

# ---------- 环境检查 ----------
command -v python3 >/dev/null 2>&1 || error "未找到 python3"

# ---------- 打印配置摘要 ----------
echo ""
echo -e "${CYAN}========== 启动配置 ==========${NC}"
echo -e "  环境:     ${GREEN}${ENV}${NC}"
echo -e "  配置文件: ${GREEN}${ENV_FILE}${NC}"
echo -e "  地址:     ${GREEN}${SERVER_HOST}:${SERVER_PORT}${NC}"
echo -e "  调试模式: ${GREEN}${DEBUG}${NC}"
echo -e "  日志级别: ${GREEN}${LOGGER_LEVEL}${NC}"
echo -e "${CYAN}==============================${NC}"
echo ""

# ---------- 根据环境选择启动方式 ----------
if [ "$ENV" = "dev" ]; then
    # 开发环境：热重载 + 详细日志
    info "启动开发服务器..."
    info "API 文档: http://${SERVER_HOST}:${SERVER_PORT}/docs"
    info "按 Ctrl+C 停止服务器"
    echo ""

    uvicorn app.main:app \
        --host "$SERVER_HOST" \
        --port "$SERVER_PORT" \
        --reload \
        --reload-dir app \
        --reload-delay 0.5 \
        --log-level debug \
        --access-log

elif [ "$ENV" = "prod" ]; then
    # 生产环境：Gunicorn + Uvicorn Worker
    WORKERS=${WORKERS:-$(($(nproc) * 2 + 1))}

    command -v gunicorn >/dev/null 2>&1 \
        || error "未安装 gunicorn，请运行 pip install gunicorn"

    info "启动生产服务器..."
    info "Worker 数量: ${WORKERS}"
    info "按 Ctrl+C 停止服务器"
    echo ""

    exec gunicorn app.main:app \
        --workers "$WORKERS" \
        --worker-class uvicorn.workers.UvicornWorker \
        --bind "${SERVER_HOST}:${SERVER_PORT}" \
        --timeout 120 \
        --graceful-timeout 30 \
        --keep-alive 5 \
        --max-requests 1000 \
        --max-requests-jitter 50 \
        --access-logfile - \
        --error-logfile - \
        --log-level "$LOGGER_LEVEL"
fi
```

### 核心：`read_env` 函数解析

`.env` 文件的格式不是标准的 shell 变量（有空格、引号、注释），不能直接 `source`。需要专门的解析函数：

```bash
# .env.dev 中的原始内容：
# SERVER_PORT = 8001          # 服务端口
# SERVER_HOST = "localhost"
# DEBUG = True

read_env() {
    local key="$1"
    local default="${2:-}"
    local value

    value=$(grep -E "^${key}\s*=" "$ENV_FILE" \  # 1. grep 匹配 key 行
        | head -1 \                                # 2. 取第一行（防止重复定义）
        | sed "s/^${key}\s*=\s*//" \               # 3. 去掉 key = 部分
        | sed 's/\s*#.*$//' \                      # 4. 去掉行尾 # 注释
        | sed 's/^["'\'']//' \                     # 5. 去掉开头引号
        | sed 's/["'\'']$//' \                     # 6. 去掉结尾引号
        | xargs)                                   # 7. 去掉首尾空格

    if [ -z "$value" ]; then
        echo "$default"
    else
        echo "$value"
    fi
}

# 使用：
SERVER_PORT=$(read_env SERVER_PORT "8001")
# 结果：SERVER_PORT=8001
```

处理流程示例：

```
原始行：  SERVER_PORT = 8001          # 服务端口
              │
grep 匹配：SERVER_PORT = 8001          # 服务端口
              │
去掉 key： = 8001          # 服务端口
              │
去掉注释： = 8001
              │
去引号：   8001
              │
去空格：   8001  ← 最终结果
```

## 8.9 Gunicorn + Uvicorn 生产部署

### 为什么生产环境用 Gunicorn

Uvicorn 可以单独用于生产，但 Gunicorn 提供了更成熟的进程管理：

```
Uvicorn 单独使用：
┌──────────────────────────┐
│      Uvicorn (单进程)     │
│      ┌──────────┐        │
│      │  Worker   │        │
│      └──────────┘        │
└──────────────────────────┘
  崩溃 → 服务终止，需要外部守护

Gunicorn + Uvicorn Worker：
┌────────────────────────────────────────┐
│           Gunicorn (Master)            │
│  负责：进程管理、信号处理、Worker 监控   │
│                                        │
│  ┌──────────┐ ┌──────────┐ ┌────────┐ │
│  │ Worker 1 │ │ Worker 2 │ │Worker 3│ │
│  │ Uvicorn  │ │ Uvicorn  │ │Uvicorn │ │
│  └──────────┘ └──────────┘ └────────┘ │
└────────────────────────────────────────┘
  Worker 崩溃 → Master 自动重启，服务不中断
```

### Gunicorn 核心参数

```bash
gunicorn app.main:app \
    --workers 4 \                          # Worker 进程数
    --worker-class uvicorn.workers.UvicornWorker \  # 使用 Uvicorn 作为 Worker
    --bind 0.0.0.0:8001 \                 # 监听地址
    --timeout 120 \                        # Worker 超时（秒），超时未响应则重启
    --graceful-timeout 30 \                # 优雅关闭超时（秒）
    --keep-alive 5 \                       # 长连接保持时间（秒）
    --max-requests 1000 \                  # 单 Worker 最大请求数，达到后重启（防内存泄漏）
    --max-requests-jitter 50 \             # 最大请求数随机抖动（避免同时重启）
    --access-logfile - \                   # 访问日志输出到标准输出
    --error-logfile - \                    # 错误日志输出到标准输出
    --log-level info                       # 日志级别
```

### Worker 数量选择

```
Worker 数量 = CPU 核心数 × 2 + 1

例如：
  2 核 CPU → 5 个 Worker
  4 核 CPU → 9 个 Worker
  8 核 CPU → 17 个 Worker

注意：
  - I/O 密集型（API 服务）：可以适当多一些
  - CPU 密集型（计算服务）：建议等于核心数
  - 内存有限时：根据单 Worker 内存消耗计算上限
```

## 8.10 systemd 进程守护（Linux 生产环境）

### 什么是 systemd

systemd 是 Linux 系统的服务管理器，负责开机自启动、进程守护、日志管理等。

### 创建 service 文件

创建 `/etc/systemd/system/fastapi-app.service`：

```ini
[Unit]
Description=FastAPI Application
After=network.target mysql.service redis.service

[Service]
User=www-data
Group=www-data
WorkingDirectory=/home/vincent/project/ai-engineer/backend
Environment="PATH=/home/vincent/project/ai-engineer/backend/.venv/bin"
Environment="ENVIRONMENT=prod"
ExecStart=/home/vincent/project/ai-engineer/backend/.venv/bin/gunicorn \
    app.main:app \
    --workers 4 \
    --worker-class uvicorn.workers.UvicornWorker \
    --bind 0.0.0.0:8001 \
    --timeout 120 \
    --max-requests 1000 \
    --max-requests-jitter 50 \
    --access-logfile - \
    --error-logfile - \
    --log-level info
ExecReload=/bin/kill -s HUP $MAINPID
ExecStop=/bin/kill -s TERM $MAINPID
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
```

```bash

sudo systemctl enable fastapi-app
```

## 8.12 应用生命周期事件

### lifespan 上下文管理器

FastAPI 的生命周期事件用于在应用启动/关闭时执行初始化和清理逻辑：

```python
from contextlib import asynccontextmanager
from fastapi import FastAPI


@asynccontextmanager
async def lifespan(app: FastAPI):
    # ===== 启动阶段 =====
    # 1. 初始化数据库连接池
    # 2. 初始化 Redis 连接
    # 3. 预加载配置到缓存
    # 4. 启动后台定时任务
    print("应用启动完成")
    yield
    # ===== 关闭阶段 =====
    # 1. 关闭数据库连接池
    # 2. 关闭 Redis 连接
    # 3. 停止后台任务
    # 4. 等待正在处理的请求完成
    print("应用已关闭")
```

### 在工厂函数中使用

```python
def create_app() -> FastAPI:
    app = FastAPI(
        title=setting.TITLE,
        lifespan=lifespan,  # 注册生命周期
    )
    return app
```

## 8.13 常见问题排查

### 问题1：`ModuleNotFoundError: No module named 'app'`

**原因**：Python 找不到 `app` 模块，通常是因为没有在项目根目录下运行。

**解决**：

```bash
# 确保在项目根目录下运行
cd /home/vincent/project/ai-engineer/backend
uvicorn app.main:app --reload

# 或设置 PYTHONPATH
export PYTHONPATH=/home/vincent/project/ai-engineer/backend:$PYTHONPATH
uvicorn app.main:app --reload
```

### 问题2：`Error: Could not import "app.main:app"`

**原因**：`main.py` 中没有 `app` 变量。

**解决**：在 `main.py` 末尾添加 `app = create_app()`，或使用 `--factory`：

```bash
uvicorn app.main:create_app --factory
```

### 问题3：`Address already in use`

**原因**：端口被其他进程占用。

**解决**：

```bash
# 查找占用端口的进程
lsof -i :8001
# 或
ss -tlnp | grep 8001

# 终止进程
kill -9 <PID>

# 或使用其他端口
uvicorn app.main:app --port 8002
```

### 问题4：热重载不生效

**原因**：文件监视目录配置不正确。

**解决**：

```bash
# 指定监视目录
uvicorn app.main:app --reload --reload-dir app

# 如果项目结构复杂，指定多个目录
uvicorn app.main:app --reload --reload-dir app --reload-dir config
```

### 问题5：`--reload` 和 `--workers` 冲突

**原因**：Uvicorn 不支持同时使用热重载和多 Worker。

**解决**：开发用 `--reload`，生产用 `--workers`，不要同时使用：

```bash
# 开发
uvicorn app.main:app --reload

# 生产
uvicorn app.main:app --workers 4
# 或用 Gunicorn
gunicorn app.main:app --workers 4 --worker-class uvicorn.workers.UvicornWorker
```
